The Questioners

Is a WEB application used for storing/creating/editing and deleting tasks and many more.

Built on a RESTful API server using Express, MongoDB and AngularJS.

It contains two main sections: To-do list and QaA(Questions and Answers).

Actively in development is a new section called Blog.

About

To-do list

Contains adding, checking, removing and permanently deleting tasks. Each task has a title, description and type with it's own defined color. Currently there are available only 3 types: work, personal and school . Checking means completing tasks, which visualy adds a line through text of the task. It is also possible to edit tasks. Tasks have 3 states: inprogress, completed and removed. It is possible to filter tasks by their type or state. Furthermore tasks have dates of their creation and completion. Deleting task permanently means deleting it off the database to never be retrieved again.

QaA

Questions and answers is what it sounds like, a bunch of questions with answers to them. Additionaly every QaA has a type, to identify where it belongs to. In the present it serves as a learning tool, that works as follows: users see the question and try to think of an answer, afterwards they can click on the question and an answer will pop up, so that they can verify if their answer was right.

Coming soon: checking by input if you have answered the question correctly or wrongly. Counter with how many times you have answered the question correctly. Ability to disable the question, so user won't see it anymore.

Flashcards

Coming soon: Cards would be taken from the QaA. Every card would have "priority", which would increase every time a user knows the answer to the card(question) and decreases when he doesn't know the answer. The user would "randomly" get these cards shown to him one by one and the priority decides how likely it is he will get the certain card. Also the user will have the ability to disable cards, so if he already knows or dislikes the card(question) he won't get it in his learning anymore.
